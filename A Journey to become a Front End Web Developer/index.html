LEARN TO LEARN

How To use The Command Line to Manage and Navigate a File System
Set the Stage
  • Mobile phone "Do not disturb"
  • Closing all web pages and applications that cause distractions.
  • Choosing the location that best works for me.
  • Clarifying what I am going to learn and why:

    I am learning how to use the terminal on my computer because it is a tool that is going to help me to work with my files in a faster and better way than a graphical interface. And as a future web developer, I really want to take advantage of these kinds of tools that are going to do my work routine better.

Reflection

Learning the commands to use the terminal on the computer, in general, was not difficult, it was like a game. The part where I struggle a little more was when I needed to move files that were inside others files to a specific location inside others files. But after playing a little around I almost got it right 80% of the times. I already feel the benefits of using the command line, it is faster and easier to look around (files, documents), copy, editing, etc., and there are a lot of things that you can do in one step in comparison with the graphical interface.

When I started to learn the command line I didn’t do a bad job in set my stage for study, but I was in a table next to other classmates and I couldn’t focus in the beginning. That made me ask for help when I really did not need. After a break a changed my location for a place where I was a long and it was quiet. There I focus really well. And it allowed me to play with the command, I repeat them until I could use them in a more “automatic way”.

Now that I figured out what is the best way for me to learn the commands to use the terminal and what is the logic in it, I am looking forward to learning a couple of commands every day.

CSS Flexible Box Layout Module
Set the Stage

CSS Flexible Box Layout Module also know as Flexbox.

I am learning Flexbox as one of the many topics that I need to know in order to define the visual style of a web page. It is a way to distribute content in a flexible way on a website and this is a very important aspect in the design of the user interface.

Play

After you understand how to think when using flexbox, that is, how the items move according to the axis that affects them, everything becomes quite easy and you feel the benefits of being able to control properties such as the size or alignment of one or a group of items at a time. It is quite a handful to use, and it gives you the possibility to do very nice things in the user interface.

(see more in the video)

Reflection

Organizing the stage to study this topic and any other is totally necessary because you need to be very concentrated to absorb new concepts or skills in the right way.

In this particular subject you have to understand the flow of the elements in their axes, and then understand all the properties with their respective values which you can apply in the organization of an item or a group of items in space. If when you are in the process of understanding it, you get distracted, the more likely it is that you have to start over, because it is very easy to lose the thread at the beginning when you do not have domain yet of which property affects which element according to which axis and all of it starts to move, extend, shrink ,etc. in the opposite direction to your desires.

With the knowledge of flexbox I am going to be able to lay out items by controlling their dimensions and the spacing between them in a very skill way that allow me to have a more creative and a faster form of doing a user interface. Now that I understand the basics of the CSS Flexible Box Layout, I can first, go deeper in the use of their properties to expand my possibilities of creating websites and second, I can compare It with CSS Grid Layout and find out the difference between them, their benefits and when is better to use each one or both.
